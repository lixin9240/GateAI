### 一、认证模块（AuthController）

| 接口路径 | 请求方式 | 变更类型 | 说明 |
|---------|---------|---------|------|
| `/api/v1/login` | POST | **修改** | 认证方式从 JWT 改为 Sanctum；`auth('api')->attempt()` 替换为 `Hash::check()` + `$user->createToken()` |
| `/api/v1/logout` | POST | **修改** | 登出方式从 JWT logout 改为 Sanctum `currentAccessToken()->delete()` |
| `/api/v1/me` | GET | **修改** | 返回字段不变，底层改用 Sanctum 认证 |

**请求参数变更（login）**：

| 参数名 | 参数类型 | 是否必传 | 说明 |
|-------|---------|---------|------|
| email | string | 是 | 邮箱 |
| password | string | 是 | 密码 |

**返回参数变更（login）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0=成功, 20001=账号或密码错误 |
| msg | string | 提示信息 |
| data.token | string | API Token |
| data.token_type | string | Bearer |
| success | bool | 是否成功 |
| trace_id | string | 链路追踪ID |

> **删除字段**: `expires_in`（Sanctum token 无过期时间返回）

### 全局变更说明

1. **统一响应格式**: 所有接口返回结构统一为 `{code, msg, data, success, trace_id}`
2. **时间字段**: 所有 `created_at`, `updated_at`, `deleted_at`, `acknowledged_at`, `executed_at`, `trigger_time`, `recover_time` 等时间字段均使用北京时间（Asia/Shanghai, UTC+8）
3. **异常处理**: Service 层统一抛出 `BusinessException`，由 `Handler::render()` 统一转换为 Result 响应
4. **链路追踪**: 所有响应均包含 `trace_id` 字段，由全局 `TraceIdMiddleware` 注入
