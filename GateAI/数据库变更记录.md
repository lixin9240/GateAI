## 2026-07-04 闸门互锁模块 + EdgeToken HMAC

**操作人**: wjc

| 变更类型 | 说明 |
|---------|------|
| 新增表 | `gate_interlock_rules` — 闸门互锁规则表，支持全局默认（reservoir_id=NULL）+ 水库专属规则，按 priority 排序执行 |
| 新增表 | `gate_interlock_logs` — 互锁触发日志表，记录每次互锁触发时的水位快照、闸门开度变化 |
| 新增字段 | `gate_actions` 新增 `interlock_rule_id` — 关联触发的互锁规则，`belongsTo` GateInterlockRule |
| 新增字段 | `edge_nodes` 新增 `api_secret` VARCHAR(64) — 边缘端 HMAC-SHA256 签名密钥 |
| 新增迁移 | `2026_07_04_100000_create_gate_interlock_rules_table.php` |
| 新增迁移 | `2026_07_04_100001_create_gate_interlock_logs_table.php` |
| 新增迁移 | `2026_07_04_100002_add_interlock_rule_id_to_gate_actions_table.php` |
| 新增迁移 | `2026_07_04_100003_add_api_secret_to_edge_nodes_table.php` |
| 数据填充 | 新增 `GateInterlockRuleSeeder` — 4 条全局默认规则（泄洪-发电互斥、下游冲击保护、对称性约束、最小下泄保障） |
| 注册 Seed | `DatabaseSeeder` 新增 `GateInterlockRuleSeeder::class` |

**表结构摘要** (`gate_interlock_rules`):

| 字段 | 类型 | 说明 |
|------|------|------|
| reservoir_id | BIGINT NULL | NULL=全局默认，有值=水库专属 |
| rule_code | VARCHAR(50) | 规则唯一标识 |
| rule_name | VARCHAR(100) | 规则中文名 |
| enabled | TINYINT(1) | 是否启用 |
| priority | INT | 优先级，数字越小越高 |
| trigger_conditions | JSON | 触发条件参数 |
| constraint_action | JSON | 约束动作参数 |

> 唯一约束：`uk_reservoir_rule` (reservoir_id + rule_code)。索引：`idx_reservoir_enabled` (reservoir_id + enabled)。

**表结构摘要** (`gate_interlock_logs`):

| 字段 | 类型 | 说明 |
|------|------|------|
| reservoir_id | BIGINT | 水库ID |
| rule_id | BIGINT | 触发的规则ID |
| decision_id | BIGINT NULL | 关联调度决策ID |
| trigger_time | TIMESTAMP | 触发时间 |
| gate1/2/3_opening_before | DECIMAL(5,4) | 互锁前开度 |
| gate1/2/3_opening_after | DECIMAL(5,4) | 互锁后开度 |
| upstream/downstream_level | DECIMAL(6,2) | 触发时水位 |
| inflow_rate | DECIMAL(8,2) | 触发时入库流量 |
| action_detail | JSON | 详细约束过程描述 |

> 索引：`idx_reservoir_time` (reservoir_id + trigger_time)，`idx_rule_id` (rule_id)。

## 2026-07-04 新增物理防护配置表

**操作人**: lx

| 变更类型 | 说明 |
|---------|------|
| 新增表 | `physics_guard_configs` — 物理防护配置表，替代 Python 侧硬编码的安全阈值（上游/下游水位线、生态流量、影子水位参数、指令平滑参数、双因子熔断阈值、闸门泄流量等），支持多水库差异化、版本追溯与回滚 |
| 新增迁移 | `2026_07_04_100001_create_physics_guard_configs_table.php` |
| 数据填充 | 新增 `PhysicsGuardConfigSeeder` — 5条配置（三峡、溪洛渡、向家坝、上游模拟、下游模拟），版本号 1.0.0，初始值从 deploy_config.json 迁移 |
| 注册 Seed | `DatabaseSeeder` 新增 `PhysicsGuardConfigSeeder::class` 调用 |

**表结构摘要** (`physics_guard_configs`):

| 字段分组 | 字段 | 默认值 |
|----------|------|--------|
| 上游水位 | upstream_danger, upstream_emergency, upstream_warning, upstream_min, ideal_min, ideal_max | 190 / 193 / 188 / 167 / 178 / 188 |
| 下游水位 | downstream_danger, downstream_max, downstream_min | 128 / 130 / 115 |
| 生态流量 | eco_flow_min | 20 |
| 物理参数 | reservoir_area, max_level_change_per_hour | 15000000 / 2.0 |
| 影子水位 | shadow_lookahead_steps, shadow_danger_offset | 3 / 3.0 |
| 指令平滑 | deadband_percent, max_rate_per_hour | 0.02 / 0.10 |
| 熔断阈值 | fusion_l3_confidence/risk, fusion_l2_confidence/risk | 0.70/0.30, 0.50/0.10 |
| 闸门参数 | gate_max_discharge (JSON) | [300, 200, 250] |
| 元数据 | config_version, is_active, description, updated_by | 1.0.0 / 1 / null / null |

> 唯一约束：`uk_reservoir_version` (reservoir_id + config_version)，同一水库同一版本号不可重复。
> 外键：`reservoir_id` → `reservoirs.id`，`updated_by` → `users.id`。
